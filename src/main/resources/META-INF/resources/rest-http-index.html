REST API with Red Hat Fuse

The REST API demonstrates how to map business operations to a remote procedure call endpoint over HTTP using a REST framework.

This project introduces the mechanics of interacting with a remote (exposed by Apache Camel) service using the HTTP protocol.

To demonstrate the behavior of this project:

  • Click camel/country/{country_name} to execute an HTTP GET request on the camel/country/{country_name} endpoint.

    A response in JSON format with some country information such as population, etc. By default, Spain is used.

  • Click camel/greetings/{name} to execute an HTTP GET request on the camel/greetings/{name} endpoint.

    A new browser window opens and shows a response in JSON format with a payload of Hello, $name! where $name is the value of the {name} URL parameter from the HTTP GET request. By default, this is Jacopo.

Click API Swagger page to open the REST API's Swagger page.
